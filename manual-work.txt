# foot-tactic 手動作業管理 (manual-work)

ルール:
- ここに記載のWorkはAIでは実行できない(人間のアカウント操作・決済・コンソール操作が必要な)作業
- 各Workには「期限」として、最悪どのPhase開始までに完了していれば開発を止めずに進められるかを記載
- AIは各Phase着手前にこのファイルを確認し、期限が到来しているWorkが未完了の場合は
  Phaseのタスクを進めず、未完了のWorkがある旨をユーザーに報告する(CLAUDE.md参照)
- 完了時は [x] と日付を記入する

[x] Work1.3 Cloud Firestoreデータベース作成 (2026-07-19 完了・本番環境モード)
    期限: Phase1.4 開始まで
    手順:
      1. Firebaseコンソール > Firestore Database > 「データベースを作成」
      2. ロケーション: asia-northeast1(東京)推奨
      3. 「本番環境モード」で開始(ルールはPhase1.6でAIがデプロイ用ファイルを作成)

[x] Work1.4 GitHubリモートリポジトリの用意 (2026-07-19 完了確認)
    期限: Phase1.1 開始まで(各タスク完了ごとにcommit+pushするルールのため)
    手順:
      1. ターミナルで `gh auth status` を実行し、GitHub CLIが認証済みか確認
         (Claude Codeのセッション内なら「! gh auth status」で実行可能)
      2. 認証済みであればリポジトリ作成(git init/リモート設定/初回push)はAIが実行する
      3. 未認証の場合は `gh auth login` を実行して認証するか、
         GitHub上で空のリポジトリ(例: foot-tactic)を作成しリモートURLをAIに共有する

[x] Work1.5 Firestoreセキュリティルールの本番反映 (2026-07-19 完了・方法B(コンソール貼り付け)で公開)
    期限: Phase2 開始まで(Phase1完了後のログイン動作確認に必要)
    手順(いずれか一方でOK):
      方法A: ターミナルで `firebase login` を実行しブラウザで認証
             (Claude Codeのセッション内なら「! firebase login」で実行可能。
              認証後のルールデプロイはAIが実行する。Work4.1も同時に完了扱いになる)
      方法B: Firebaseコンソール > Firestore Database > ルール タブを開き、
             リポジトリの firestore.rules の内容を貼り付けて「公開」を押す
    備考: 本番環境モードの初期ルールは全拒否のため、反映するまで
          ログイン時のユーザードキュメント作成が失敗する(ログイン自体は可能)

Work3: Googleドキュメント保存用のGoogle Cloud設定
=====================================================
[x] Work3.1 Docs API / Drive APIの有効化とOAuth同意画面の設定 (2026-07-20 完了・スクリーンショット確認済み。公開ステータス=本番環境)
    期限: Phase7.1 開始まで
    手順:
      1. https://console.cloud.google.com/ でFirebaseと同じGCPプロジェクトを開く
      2. 「APIとサービス > ライブラリ」から以下を有効化:
         - Google Docs API
         - Google Drive API
      3. 「APIとサービス > OAuth同意画面」で以下を設定:
         - アプリ名・サポートメール・デベロッパー連絡先
         - スコープに documents / drive.file を追加
         - 公開ステータスが「テスト中」の場合、テストユーザーに自分のGoogleアカウントを追加
    備考: OAuthクライアントIDはFirebase AuthのWebクライアントを流用可能。
          別途必要になった場合はAIが指示する
