# foot-tactic 実装タスク管理 (progress)

ルール:
- 各タスクはSonnet/Opusで問題なく実装できる粒度に分解している
- 各Phase開始前に manual-work.txt を確認し、期限が到来しているWorkが未完了の場合は
  そのPhaseに着手せず、ユーザーに未完了Workを報告すること(CLAUDE.md参照)
- 各タスク完了時: テスト作成・実行 + `npx tsc --noEmit` を通してから [x] にする
- 各タスク(Phase X.Y)完了ごとに必ずgitへコミットしpushする
- 1つのPhaseの全タスクが終了したら必ず作業を止めてユーザーの確認を求める。
  問題ないことが確認できるまで次のPhaseに進まない(CLAUDE.md参照)
- 完了時は日付を記入する 例: [x] (2026-07-19)

    - APIキー登録UI(localStorageのみに保存、サーバー非送信の明示、キーのマスク表示)
    - 利用料はユーザーのAPIアカウントに課金される旨の注意文言(日/英)
[ ] Phase6.2 録音機能
    - MediaRecorderによる録音ボタン(録音中表示、停止)
    - マイク権限拒否時のエラー表示
    - APIキー未登録時は設定画面へ誘導
[ ] Phase6.3 OpenAI文字起こし連携【実装のみ・テスト対象外】
    - 録音音声をOpenAI Audio API(gpt-4o-transcribe/Whisper)へ直接送信する処理の実装
    - 認証エラー/ネットワークエラーのハンドリング(日/英)
    - ※APIキーが用意できないため、テスト作成・実キーでの動作検証は行わない
    - ※型チェック(npx tsc --noEmit)のみ通すこと。動作検証はPhase6.4のGeminiで行う
[ ] Phase6.4 Gemini文字起こし連携
    - 録音音声をGemini APIへ直接送信、エラーハンドリング
[ ] Phase6.5 文字起こし結果の挿入フロー
    - プレビュー表示→採用/編集/破棄
    - 採用時に新規メモエントリ(source=voice)として挿入、タグ付与可能
[ ] Phase6.6 API無料枠超過の検知とブロック機能
    - 文字起こしAPIのクォータ超過エラー(429/RESOURCE_EXHAUSTED等)を検知
    - 超過検知時は「無料枠を使い切りました」の通知を表示(日/英)
    - 超過後はAPI呼び出しをブロックし録音ボタンを無効化する
    - クォータは日次リセットされるため、ユーザーが手動で再有効化できる導線を用意
    - ブロック状態はlocalStorageに保持しリロード後も維持する
